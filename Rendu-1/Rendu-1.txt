1 - Les éléments entourés en vert sont récupérés de façons dynamique depuis le fichier .json. Pour la responsivité, j'ai dû retirer certains éléments qui sont GHST, LesScoresDeChantal; pour que la responsitivté soit parfaite surtout sur mobile car 95% du trafic sur internet se fait sur mobile.

2 - Les éléments entourés en noir (les images) sont récupérés de façon statiques, c'est-à-dire qu'ils en sont pas récupérer depuis le fichier .json. J'ai dû improviser comme cela car, au vue des deux images, j'ai supposé que ce sont des données qui ne changeront pas très souvent. Aussi dropbox ne permet pas un accès direct aux données utilisateurs, c'est-à-dire qu'il empêche la récupération des images via l'url. Pour palier a ce problème, j'ai télécharger les images sur mon ordinateur, importé ces dernières dans le dossier assets ( situé dans le code ). De ce dernier dossier, je les importé dans le dossier data ( situé dans le code et contenant le fichier index.jx ), dans lequel je leurs ai assigné des variables ( priceImg pour la photo de price, twitterImg pour la photo de twitter ) que j'ai à leur tour assigné aux variables ( priceImgUrl pour priceImg et twitterImgUrl pour twitterImg ). De là j'ai pu les récupérer dans mon code. Toutefois, pour modifier cela, il suffit juste de modifier les images dans le dossier assets puis dans le fichier index.js du dossier data. Les liens, quant à eux, sont récupérer depuis le fichier .json. 

Ceci est le premier aperçu du rendu, si vous tenez à recupérer les images directement depuis le fichier .json, faites-le moi savoir car je fais des recherches sur les moyens possibles existant pour contourner le pare-feu de dropbox.

3 - Le rendu pour la page complète n'est pas identique à celle de l'image car les deux images cliquables n'ont pas les mêmes dimensions. Toutefois, je peux les ramenées à la même dimensions mais pour la responsivité surtout pour les utilisateurs de mobile, le rendu ne serait pas optimale. Faites le moi savoir si cela vous convient ou pas.

4 - J'ai ajouté une petite animation lorsque les données sont en cours de récupération depuis le fichier .json. Pendant le léger temps vide qui sépare la récupération des données et leurs affichages, j'ai ajouté une petite animation d'un demi-cercle qui tourne, afin de notifier à l'utilisateur que les données dont en cours de téléchagement. Ne vous en faites pas, ce laps de temps passe presque inaperçu, l'utilsateur n'aura même pas le temps de s'en rendre compte qu'il searit déjà passé. Mais pour un site professionnel, il le faut; surtout quand il y a beaucoup de personnes actives au même moment sur la même page. 

Donnez moi vos avis sur ce premier rendu et les modifications que vous aimeriez ajouter.
